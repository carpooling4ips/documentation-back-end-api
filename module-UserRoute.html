# route User Search

Route:
MethodPath
GET /user/:user/search
Search for the specified user.
Authentication

This route requires a valid user's token. If authentication fails it will return a 401 http status code.

Header Parameters:
Name Type Description
Authorization string The authentication user's token.
Query Parameters:
Name Type Description
query string The user searching filter parameter string.
Route Parameters:
Name Type Description
:user string Use 'me' to reference the logged-in user or specify a user's id to act on the behalf of that user.
Response:
Name Type Attributes Description
id string The filtered user's id.
name string The filtered user's name.
title string The filtered user's title.
avatar.fileName string optional The filtered user's avatar file name on the server.
avatar.mimeType string optional The filtered user's avatar file mime type.
Response Code:
Type Description
200 The searching users retrieval succeeded.
401 The user authentication has failed.
403 The authenticated user does not have permission.
404 The requested user id does not exist.
422 The request content is unable to be processed due to invalid fields.
500 An internal server error.
